# 자주 묻는 질문

[한국어 문서 목차](./README.md) · [시작하기](./getting-started.md) · [Windows 설치](./windows-installation.md)

## 기본 개념

### PPT Master는 웹 서비스인가요?

아닙니다. PPT Master는 AI 에이전트가 로컬 파일과 스크립트를 사용해 프레젠테이션을 만드는 워크플로입니다. 에이전트와 모델 API 통신을 제외한 파일 처리와 내보내기는 기본적으로 사용자 환경에서 수행됩니다.

### 결과물이 정말 편집 가능한가요?

SVG의 텍스트와 도형을 PowerPoint DrawingML 요소로 변환하므로 대부분의 텍스트·도형·표·차트를 개별 선택하고 편집할 수 있습니다. 다만 복잡한 필터, 지원되지 않는 SVG 효과, 사진은 래스터 이미지로 남을 수 있습니다.

### 한 번의 요청으로 완성본이 나오나요?

자료와 모델, 지시의 품질에 따라 다릅니다. 초안 생성과 반복 작업을 크게 줄여 주지만 중요한 발표 자료는 사실 검증, 문구 다듬기, 레이아웃 확인이 필요합니다.

### 어떤 모델을 사용해야 하나요?

긴 문서를 안정적으로 읽고 파일·명령 도구를 정확히 사용하는 모델이 유리합니다. 모델이 작거나 컨텍스트가 짧으면 슬라이드 간 일관성, 출처 보존, 시각 검토가 약해질 수 있습니다.

## 설치와 실행

### Python 외에 무엇이 필요한가요?

일반적인 PDF/DOCX/이미지 입력은 `requirements.txt` 설치로 충분합니다. 일부 레거시 문서 형식은 Pandoc이 필요할 수 있고, 이미지 생성·검색·TTS 공급자는 별도 API 키가 필요할 수 있습니다.

### `python`과 `python3` 중 무엇을 써야 하나요?

환경에 설치된 명령을 사용하면 됩니다. Windows는 `python` 또는 `py`, macOS/Linux는 `python3`인 경우가 많습니다. 의존성 설치는 다음처럼 인터프리터를 명시하면 안전합니다.

```bash
python -m pip install -r requirements.txt
```

### 에이전트가 PPT Master를 모르는 것처럼 행동합니다

다음처럼 핵심 워크플로를 다시 읽게 하세요.

```text
skills/ppt-master/SKILL.md를 읽고 현재 프로젝트 상태를 확인한 뒤 계속해 줘.
```

### 업데이트 후 동작이 달라졌습니다

`git status`, 현재 브랜치, 원격 저장소를 확인하세요. 이 포크의 한국어 변경을 유지하려면 `origin`이 `Leafmire/ppt-master`를 가리켜야 합니다.

```bash
git status
git branch --show-current
git remote -v
```

## 한국어와 글꼴

### 한글이 중국어 글꼴로 표시되는 이유는 무엇이었나요?

원본의 초기 텍스트 런 언어 판별은 CJK 문자를 하나의 `zh-CN` 경로로 처리했습니다. PowerPoint는 DrawingML의 언어와 `latin`/`ea` 글꼴 슬롯을 함께 사용하므로, 한글도 중국어 EA 기본 글꼴로 연결될 수 있었습니다.

이 포크는 한글을 별도로 감지해 `ko-KR`로 기록하고 한국어용 EA 글꼴을 선택합니다.

### 어떤 한글 문자를 감지하나요?

- 한글 완성형 음절
- 한글 자모
- 호환 자모
- 반각 한글 자모
- 자모 확장 A/B

### 기본 한국어 글꼴은 무엇인가요?

- 산세리프 계열: `Malgun Gothic`
- 세리프 계열: `Batang`

SVG에 한국어 글꼴을 명시하면 해당 스택을 우선 해석합니다.

### Noto Sans KR이나 나눔고딕을 그대로 사용하지 않는 이유는 무엇인가요?

PPTX가 Windows PowerPoint에서 열릴 가능성을 우선해 일부 macOS/Linux/웹 글꼴 이름을 Windows 기본 글꼴로 정규화합니다. 원본 글꼴을 반드시 유지해야 한다면 대상 PC에 글꼴을 설치하고 변환 매핑 정책을 프로젝트 요구에 맞게 조정해야 합니다.

### 한 슬라이드에 한국어와 영어가 섞여도 되나요?

됩니다. 언어는 텍스트 런 단위로 판별합니다. 한국어가 포함된 런은 `ko-KR`, 순수 영문 런은 `en-US`를 사용합니다. 서로 다른 언어를 별도 `<tspan>`이나 텍스트 런으로 나누면 글꼴 제어가 더 명확해집니다.

### 한국어 줄바꿈이 이상합니다

다음을 확인하세요.

- SVG 텍스트 상자의 폭이 충분한가
- 지정 글꼴이 실제 환경에 설치되어 있는가
- PowerPoint에서 대체 글꼴이 적용되지 않았는가
- 너무 긴 단어·URL·숫자열이 있는가
- 글꼴 크기와 줄 높이가 SVG 설계와 맞는가

이 포크는 한글을 CJK 전각 폭으로 추정하고 한글 경계의 줄 합치기에서 불필요한 공백을 넣지 않도록 처리합니다.

## 품질과 레이아웃

### 텍스트가 슬라이드 밖으로 넘칩니다

- 문장을 줄이거나 슬라이드를 나누기
- 글꼴 크기만 무작정 줄이지 말고 정보 구조 재설계
- 텍스트 상자의 폭·높이와 줄 높이 확인
- SVG 미리보기와 PowerPoint 렌더링을 둘 다 확인
- 대상 글꼴 설치 여부 확인

### PowerPoint에서 SVG 미리보기와 다르게 보입니다

브라우저 SVG 렌더러와 PowerPoint DrawingML 렌더러는 동일하지 않습니다. 필터, 블렌드, 마스크, 복잡한 텍스트 배치, 미지원 CSS는 차이가 날 수 있습니다. 중요한 요소는 단순한 도형과 명시적 속성으로 설계하세요.

### 모든 슬라이드가 비슷해 보입니다

에이전트에게 슬라이드 역할별 레이아웃 변화를 명시하세요.

```text
표지, 핵심 수치, 비교, 프로세스, 사례, 결론 슬라이드의 구성을 서로 다르게 하고,
같은 카드 그리드를 연속 두 장 이상 반복하지 말아 줘.
```

### 자료의 숫자가 바뀌었습니다

AI가 요약하는 과정에서 값이 변형될 수 있습니다. 요청에 다음 조건을 추가하고 최종 결과를 원본과 대조하세요.

```text
모든 수치, 날짜, 단위, 고유명사는 원문에서 그대로 인용하고,
추론하거나 반올림한 값은 별도로 표시해 줘.
```

### 출처는 어떻게 처리하나요?

원문 페이지, URL, 보고서명, 이미지 라이선스 정보를 가능한 한 슬라이드 주석이나 발표자 노트에 남깁니다. 웹 이미지 검색에서 저작자 표시가 필요한 자산을 선택했다면 슬라이드에 짧은 크레딧을 추가해야 합니다.

## 템플릿

### 기존 회사 PPTX를 재사용할 수 있나요?

가능합니다. 템플릿과 새 자료를 함께 제공하고 사용할 슬라이드·레이아웃 범위를 지정하세요. 자세한 내용은 [템플릿 사용 가이드](./templates-guide.md)를 참고하세요.

### 템플릿의 모든 요소가 자동으로 편집 가능한가요?

기존 PPTX 내부 요소는 원본 구조에 따라 다릅니다. 그림으로 들어간 배경은 그림으로 유지되며, 텍스트·표·차트 자리표시는 워크플로가 지원하는 범위에서 새 콘텐츠로 채울 수 있습니다.

### 브랜드 글꼴이 없는 PC에서 어떻게 되나요?

PowerPoint가 대체 글꼴을 사용해 줄바꿈과 레이아웃이 달라질 수 있습니다. 배포 전 대상 환경에 글꼴을 설치하거나, 안전한 대체 글꼴을 템플릿 명세에 함께 정의하세요.

## 내보내기

### 생성된 PPTX가 열리지 않습니다

1. 내보내기 로그의 첫 번째 오류 확인
2. `exports/`의 최종 파일인지 확인
3. 파일 크기가 비정상적으로 0에 가깝지 않은지 확인
4. 같은 프로젝트의 SVG 백업으로 다시 내보내기
5. PowerPoint 2016 이상에서 확인
6. 문제가 재현되는 최소 SVG를 분리해 변환 단계 확인

### 슬라이드가 이미지 한 장처럼 보입니다

네이티브 변환이 아니라 SVG 스냅샷 출력이나 미리보기 파일을 열었을 수 있습니다. 파일 이름과 내보내기 옵션을 확인하고 네이티브 PPTX를 여세요.

### LibreOffice에서도 동일하게 보이나요?

PowerPoint와 LibreOffice는 DrawingML 해석이 다를 수 있습니다. 최종 배포 대상이 PowerPoint라면 PowerPoint를 기준으로 검수하세요. LibreOffice 호환이 필수라면 별도 검증이 필요합니다.

## 이미지와 오디오

### 이미지 API 키가 없어도 되나요?

사용자 제공 이미지와 키 없는 웹 검색 경로만으로도 작업할 수 있습니다. 더 넓은 이미지 선택이나 AI 생성이 필요하면 `.env`에 공급자 키를 설정하세요.

### 내레이션을 넣으려면 무엇이 필요한가요?

발표자 노트와 TTS 백엔드 설정이 필요합니다. [오디오 내레이션 가이드](./audio-narration.md)를 참고하세요.

### 한국어 TTS 음성은 어떻게 고르나요?

사용하는 TTS 공급자에서 `ko-KR` 음성을 선택합니다. 음성 이름과 옵션은 백엔드별로 다르므로 해당 스크립트의 `--help`와 음성 목록 명령을 확인하세요.

## 도움을 요청할 때 포함할 정보

문제를 재현하기 쉽게 다음 정보를 정리하세요.

- 운영체제와 PowerPoint 버전
- Python 버전
- 현재 커밋 또는 브랜치
- 실행 명령
- 오류 로그의 첫 부분
- 문제가 발생한 최소 SVG/PPTX 또는 프로젝트 구조
- 사용한 글꼴 이름
- 기대 결과와 실제 결과
