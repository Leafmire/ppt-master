# 템플릿 사용 가이드

[한국어 문서 목차](./README.md) · [템플릿 아키텍처](./templates-architecture.md) · [FAQ](./faq.md)

PPT Master는 자유 디자인뿐 아니라 기존 PPTX, 브랜드 자산, SVG 레이아웃을 재사용할 수 있습니다. 이 문서는 템플릿을 안전하게 준비하고 에이전트에 전달하는 방법을 설명합니다.

## 템플릿 방식 선택

### 기존 PPTX 채우기

회사·학교·고객의 실제 PowerPoint 템플릿을 유지해야 할 때 적합합니다.

```text
projects/templates/company-template.pptx를 사용해
projects/report/sources의 내용을 채워 줘.

- 원본 슬라이드 마스터와 테마 유지
- 표지, 목차, 본문, 비교, 결론 레이아웃만 사용
- 텍스트, 표, 차트는 편집 가능한 상태 유지
- 숨김 슬라이드와 예시 데이터는 결과에서 제외
```

### PPTX를 시각 참조로 사용

기존 PPTX의 분위기만 참고하고 새 레이아웃을 만들고 싶을 때 사용합니다.

```text
reference.pptx의 색상, 타이포그래피, 여백 감각만 참고하고
슬라이드 구조는 새 콘텐츠에 맞게 다시 설계해 줘.
```

### SVG 레이아웃 템플릿

반복 가능한 레이아웃을 SVG로 관리할 때 적합합니다. 텍스트·도형 구조를 직접 제어할 수 있고 네이티브 변환 경로와 잘 맞습니다.

### 브랜드 템플릿 패키지

`skills/ppt-master/templates/brands/` 아래의 디자인 명세, 자산, 레이아웃을 묶어 재사용할 수 있습니다.

## 템플릿 준비 체크리스트

기존 PPTX를 제공하기 전에 다음을 정리하면 결과가 안정적입니다.

- 실제 사용할 슬라이드 레이아웃만 남기거나 번호를 지정
- 예시 문구와 실제 고정 문구 구분
- 로고·법적 고지·페이지 번호 중 삭제하면 안 되는 요소 표시
- 브랜드 색상 HEX/RGB 값 정리
- 제목·본문·캡션의 최소 글꼴 크기 정의
- 브랜드 글꼴과 안전한 대체 글꼴 명시
- 차트 색상 순서와 표 스타일 정의
- 16:9, 4:3 등 캔버스 비율 확인
- 숨김 슬라이드와 사용하지 않는 마스터 정리

## 한국어 템플릿 권장 글꼴

여러 Windows PC에서 열어야 한다면 다음 조합이 안전합니다.

| 역할 | 권장 글꼴 | 대체 |
|---|---|---|
| 한국어 제목 | Malgun Gothic Bold | Malgun Gothic |
| 한국어 본문 | Malgun Gothic | Gulim / Dotum |
| 한국어 세리프 | Batang | BatangChe |
| 영문 UI 계열 | Segoe UI | Arial |
| 영문 세리프 | Times New Roman | Cambria |

브랜드 전용 한국어 글꼴을 사용하는 경우 대상 PC 설치 여부를 확인하고, 템플릿 명세에 대체 글꼴을 함께 기록하세요.

SVG 예시:

```svg
<text font-family="Brand Sans KR, Noto Sans KR, Malgun Gothic, sans-serif">
  한국어 제목
</text>
```

이 포크는 일반적인 한국어 글꼴 이름과 로컬라이즈된 이름을 인식하고 Windows 호환 계열로 정규화합니다.

## 에이전트에 전달할 정보

템플릿 파일만 던지는 것보다 다음 정보를 함께 주는 편이 좋습니다.

```text
템플릿: projects/templates/company-template.pptx
자료: projects/strategy/sources/
청중: 한국 지사 임원
형식: 16:9, 12장
사용 레이아웃: 원본 1, 3, 5, 7번 계열
금지: 로고 위치 변경, 배경색 변경, 16pt 미만 본문
글꼴: 한국어 맑은 고딕, 영문 Segoe UI
차트: 브랜드 팔레트 순서 유지
```

## 콘텐츠를 템플릿에 맞추는 원칙

### 레이아웃에 정보를 억지로 넣지 않기

콘텐츠가 많으면 글꼴을 과도하게 줄이기보다 슬라이드를 나누거나 다른 레이아웃을 선택합니다.

### 자리표시자의 의미 유지

제목 자리에는 제목, 본문 자리에는 본문을 넣습니다. 장식용 도형이나 로고를 콘텐츠 자리로 오해하지 않도록 에이전트가 템플릿 구조를 먼저 분석해야 합니다.

### 데이터 요소는 네이티브로 유지

표와 차트를 이미지로 붙이지 말고 가능한 범위에서 PowerPoint 편집 요소로 유지하도록 요청합니다.

### 고정 요소와 가변 요소 구분

- 고정: 로고, 법적 고지, 배경, 페이지 번호 위치
- 가변: 제목, 본문, 이미지, 차트 데이터, 강조 수치

## 템플릿 분석 절차

권장 흐름:

1. PPTX의 슬라이드 크기와 마스터 확인
2. 레이아웃별 자리표시자와 고정 요소 목록화
3. 테마 색상과 글꼴 확인
4. 재사용 가능한 레이아웃 선정
5. 새 콘텐츠를 슬라이드 역할에 매핑
6. 샘플 1~2장 생성 후 시각 검토
7. 전체 생성
8. PowerPoint에서 마스터·글꼴·정렬 검수

## 자유 디자인과 혼합하기

템플릿의 표지와 결론은 유지하면서 본문 일부를 자유 디자인 SVG로 만들 수 있습니다. 다만 한 파일 안에서 두 디자인 체계가 충돌하지 않도록 다음을 통일하세요.

- 캔버스 크기
- 기본 여백
- 제목 위치와 크기
- 색상 팔레트
- 글꼴 계층
- 페이지 번호와 로고 규칙

## 템플릿 디렉터리

대표 경로:

```text
skills/ppt-master/templates/
├── brands/
├── charts/
├── decks/
├── layouts/
└── design_spec_reference.md
```

실제 구조와 사용법은 현재 저장소의 디렉터리 및 [템플릿 아키텍처](./templates-architecture.md)를 함께 확인하세요.

## 검수 체크리스트

### 디자인

- 마스터 배경과 로고가 유지되는가
- 제목·본문 계층이 원본과 일치하는가
- 브랜드 색상 외의 임의 색상이 추가되지 않았는가
- 정렬선과 여백이 일관적인가

### 콘텐츠

- 모든 수치와 단위가 원본과 일치하는가
- 자리표시자 예시 문구가 남아 있지 않은가
- 잘린 텍스트와 빈 자리표시자가 없는가
- 표·차트의 범례와 주석이 완전한가

### 한국어

- 한글 런이 올바른 글꼴로 보이는가
- 대체 글꼴 때문에 줄바꿈이 바뀌지 않았는가
- 조사와 띄어쓰기, 문장부호가 자연스러운가
- 영문·숫자와 한글의 기준선이 어색하지 않은가

## 흔한 실패와 대응

### 템플릿을 그대로 복제했지만 내용이 넘침

레이아웃을 바꾸거나 슬라이드를 추가합니다. 템플릿의 예시 문구 길이는 실제 자료보다 짧을 수 있습니다.

### 로고나 배경이 사라짐

마스터 요소인지 슬라이드 개별 요소인지 확인하고, 삭제 금지 요소를 명시합니다.

### 글꼴이 달라짐

대상 PC에 브랜드 글꼴을 설치하거나, 템플릿과 SVG에 안전한 대체 글꼴 스택을 정의합니다.

### 차트가 이미지로 들어감

요청에 “차트 데이터를 PowerPoint에서 수정 가능하게 유지”를 명시하고, 지원되는 네이티브 차트 경로를 사용하도록 합니다.

### 레이아웃 번호가 달라짐

템플릿 편집 과정에서 슬라이드와 레이아웃 순서가 바뀔 수 있습니다. 번호뿐 아니라 레이아웃 이름과 시각적 역할을 함께 설명하세요.
