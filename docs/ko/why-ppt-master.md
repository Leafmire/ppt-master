# 왜 PPT Master인가

[한국어 문서 목차](./README.md) · [시작하기](./getting-started.md) · [기술 설계](./technical-design.md)

## 핵심 차이: 결과물이 실제 PowerPoint인가

많은 AI 프레젠테이션 도구는 보기 좋은 결과를 빠르게 만들지만, 최종 산출물이 다음 중 하나인 경우가 많습니다.

- 고정 템플릿의 일부 영역만 채운 PPTX
- 슬라이드마다 이미지 한 장을 넣은 PPTX
- 웹 브라우저에서만 재생되는 HTML 프레젠테이션
- 플랫폼 안에서만 수정할 수 있는 독점 형식

PPT Master는 텍스트·도형·표·차트를 PowerPoint DrawingML 요소로 내보내는 것을 목표로 합니다. 따라서 사용자는 결과를 받은 뒤에도 PowerPoint에서 직접 수정하고 조직의 기존 업무 흐름에 연결할 수 있습니다.

| 기준 | 이미지 기반 생성기 | 웹 프레젠테이션 | 고정 템플릿 채우기 | PPT Master |
|---|---:|---:|---:|---:|
| PowerPoint 파일 | 일부 | 아니오 | 예 | 예 |
| 요소별 편집 | 아니오 | 아니오 | 제한적 | 예 |
| 로컬 파일 중심 | 도구별 상이 | 도구별 상이 | 도구별 상이 | 기본 방향 |
| 기존 PPTX 템플릿 | 제한적 | 보통 불가 | 핵심 기능 | 지원 |
| SVG 작업본 | 보통 없음 | 내부 전용 | 보통 없음 | 보존 가능 |
| 에이전트·모델 선택 | 서비스 종속 | 서비스 종속 | 서비스 종속 | 교체 가능 |

## 에이전트 워크플로라는 의미

PPT Master 자체는 완전한 AI 모델이 아닙니다.

```text
워크플로(harness) + 모델 + 도구 실행 환경 = 프레젠테이션 에이전트
```

워크플로는 해야 할 순서와 품질 검사를 정의하고, 모델은 콘텐츠 이해와 설계 판단을 담당합니다. 같은 저장소를 사용해도 모델, 프롬프트, 자료 정리 상태, 사용자의 검수 능력에 따라 결과가 달라집니다.

## 장점

### 1. 수정 가능한 결과

완성본을 다시 생성하지 않고도 다음 작업을 할 수 있습니다.

- 문구와 숫자 수정
- 도형 색·크기·위치 변경
- 차트 데이터 교체
- 회사 템플릿에 맞춘 최종 다듬기
- 발표자 노트 추가
- 슬라이드 순서 변경

### 2. 투명한 작업 과정

중간 SVG, 디자인 명세, 백업, 내보내기 로그를 확인할 수 있습니다. 문제가 생겼을 때 전체를 다시 만드는 대신 특정 슬라이드나 변환 단계만 수정할 수 있습니다.

### 3. 플랫폼 종속 완화

특정 프레젠테이션 SaaS에만 저장되는 문서가 아니라 표준 `.pptx`를 결과로 사용합니다. AI 에이전트와 모델도 하나로 고정되지 않습니다.

### 4. 로컬 자료 처리

원본 문서와 생성 파일을 로컬 프로젝트 디렉터리에서 관리할 수 있습니다. 단, 사용하는 모델/API로 전송되는 내용은 해당 공급자의 정책을 따르므로 민감 자료는 조직 규정을 먼저 확인해야 합니다.

### 5. 기존 자산 재사용

회사 템플릿, 로고, 이미지, 이전 프레젠테이션을 함께 제공해 새 콘텐츠를 기존 디자인 체계에 맞출 수 있습니다.

## 한계

### 1. 품질은 모델과 사용자에게 크게 의존

자료가 불완전하거나 요청이 모호하면 결과도 모호해집니다. 중요한 수치와 문장은 사람이 원문과 대조해야 합니다.

### 2. PowerPoint와 SVG 렌더링은 완전히 같지 않음

브라우저에서 보이는 모든 SVG 효과가 DrawingML로 동일하게 변환되지는 않습니다. 복잡한 필터, 마스크, 블렌드, 웹 전용 CSS는 단순화되거나 이미지로 처리될 수 있습니다.

### 3. 글꼴 환경 차이

대상 PC에 글꼴이 없으면 PowerPoint가 대체 글꼴을 사용합니다. 이 포크는 한국어 런에 `ko-KR`과 한국어 기본 글꼴을 지정하지만, 브랜드 전용 글꼴의 설치 여부까지 보장하지는 않습니다.

### 4. 완전 자동 사실 검증 도구가 아님

AI는 숫자, 날짜, 출처를 잘못 요약할 수 있습니다. 최종 발표 전 검증 책임은 사용자에게 있습니다.

### 5. 학습 곡선

파일 구조, 에이전트 지시, 디자인 명세, 시각 검토를 이해할수록 결과가 좋아집니다. 클릭 한 번짜리 SaaS보다 초기 설정이 필요하지만, 대신 통제 범위가 넓습니다.

## 다른 도구와 함께 쓰기

PPT Master는 기존 도구를 모두 대체할 필요가 없습니다.

- 문서 정리: 워드프로세서·노트 도구
- 데이터 검증: 스프레드시트·BI 도구
- 이미지 생성: 전용 이미지 모델
- 마지막 정리: PowerPoint
- 배포: PDF, 동영상, LMS, 회의 플랫폼

PPT Master는 이들 사이에서 자료를 분석하고 편집 가능한 슬라이드 초안을 만드는 역할에 적합합니다.

## 적합한 사용 사례

- 보고서·논문·제안서를 발표 자료로 변환
- 반복되는 월간·분기 보고서
- 회사 템플릿을 사용하는 내부 보고
- 강의 자료와 발표자 노트 작성
- 데이터가 많은 차트 중심 프레젠테이션
- 생성 후 PowerPoint에서 계속 수정해야 하는 작업

## 덜 적합한 사용 사례

- 한 장의 소셜 미디어 이미지가 최종 결과인 경우
- 브라우저 전용 인터랙티브 웹 프레젠테이션
- 검수 없이 즉시 외부에 배포해야 하는 고위험 자료
- 모든 복잡한 SVG 효과를 픽셀 단위로 동일하게 보존해야 하는 경우

## 한국어 포크의 추가 가치

이 포크는 원본의 텍스트 런별 언어 판별을 한국어까지 확장합니다.

- 한글 런을 `ko-KR`로 기록
- 한국어용 East Asian 글꼴 슬롯 분리
- 맑은 고딕/바탕 기본값
- 자모와 확장 자모 감지
- 한글 폭과 줄 합치기 처리 개선
- 한국어 설치·사용·기술 문서 제공

이는 단순 번역뿐 아니라 PowerPoint 내부의 언어·글꼴 메타데이터가 한국어 텍스트에 맞게 생성되도록 하는 변경입니다.

## 결론

PPT Master의 가치는 “AI가 슬라이드 이미지를 만들어 준다”가 아니라, **AI가 반복 작업을 줄이고 사용자가 계속 소유·수정할 수 있는 PowerPoint 작업물을 남긴다**는 데 있습니다. 빠른 초안이 필요하면서도 최종 통제권을 포기하고 싶지 않은 사용자에게 적합합니다.
