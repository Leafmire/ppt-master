# 로드맵

[한국어 문서 목차](./README.md) · [원문 로드맵](../roadmap.md)

> PPT Master와 이 한국어 포크는 고정된 출시 일정보다 실제 사용 우선순위에 따라 유지보수됩니다. 이 문서는 방향을 공유하기 위한 것이며 특정 기능의 완료 시점을 약속하지 않습니다.

## 현재 기준

핵심 파이프라인:

```text
자료 입력
→ 구조화된 텍스트 변환
→ 디자인 명세
→ SVG 생성
→ 품질 검사
→ DrawingML 변환
→ 네이티브 편집형 PPTX
```

한국어 포크의 핵심 추가 범위:

- `README_KO.md`
- `docs/ko/` 한국어 문서
- 텍스트 런별 `ko-KR` 언어 판별
- 한국어용 East Asian 글꼴 슬롯
- 맑은 고딕/바탕 기본값
- 자모·호환 자모·확장 자모 감지
- 한국어 텍스트 폭 및 줄 경계 처리

## 원본 프로젝트의 최근 방향

원본 로드맵은 다음 축을 중심으로 확장되고 있습니다.

- SVG → DrawingML 네이티브 PPTX 변환 정확도
- PowerPoint, Keynote, LibreOffice, WPS 간 렌더링 안정성
- 소스 문서 변환 충실도
- Brand / Layout / Deck 템플릿 구조
- 시각 스타일과 내러티브 모드 분리
- 브라우저 기반 미리보기와 직접 편집
- 차트·아이콘·이미지 생성 체계
- 발표자 노트, TTS 내레이션, 동영상 내보내기
- 개체 애니메이션과 전환

상세 내역과 최신 상태는 [영문 로드맵](../roadmap.md) 및 커밋 로그가 기준입니다.

## 한국어 포크의 우선순위

### P0 — 한국어 변환 안정성

- 한글 완성형과 자모의 `ko-KR` 판별 유지
- 한국어 런이 중국어 기본 글꼴로 떨어지지 않도록 회귀 확인
- 영문 `en-US`, 중국어 `zh-CN` 기존 동작 보존
- 한국어 글꼴 이름과 로컬라이즈 별칭 정규화
- PowerPoint에서 글꼴 대체 시 줄바꿈 변화 점검
- 한글 텍스트 상자 폭과 문단 병합 정확도 개선

검증 대상 예시:

```text
안녕하세요
한글 자모
2026 한국 시장
한국어 + English + 中文
맑은 고딕 / Noto Sans KR / Apple SD Gothic Neo
바탕 / Noto Serif KR / 나눔명조
```

### P0 — 한국어 문서 동기화

- 루트 README의 핵심 설치·사용 흐름 반영
- `docs/ko`와 영문 문서 간 깨진 링크 방지
- 새 CLI 옵션과 파일 경로 갱신
- Windows 한국어 환경 문제 해결 사례 축적
- 예제 명령과 출력 경로를 현재 코드에 맞게 유지

### P1 — 한국어 템플릿과 예제

- 한국어 경영진 보고서 예제
- 한국어 강의·교육 자료 예제
- 한국어 데이터 저널리즘 예제
- 공공기관·학교 환경에 안전한 기본 글꼴 조합
- 한글 제목·본문·캡션의 권장 크기와 행간 자료
- 한국어 Brand 템플릿 작성 예시

예제는 단순 번역보다 한국어 문장 길이, 조사, 줄바꿈, 숫자·단위 표기까지 검증할 수 있어야 합니다.

### P1 — 한국어 오디오

- `ko-KR` 음성 선택 경로 확인
- 숫자, 날짜, 통화, 영문 약어 발음 가이드
- 발표자 노트의 한국어 문장 분할 권장안
- 내레이션과 자동 애니메이션 타이밍 검증
- 공급자별 한국어 음성 샘플 비교 절차

### P2 — 글꼴 정책 설정 가능성

현재 기본값은 Windows PowerPoint 호환성을 우선합니다. 향후 사용 요구가 충분하면 다음과 같은 명시적 정책을 검토할 수 있습니다.

```text
windows-safe   → 맑은 고딕 / 바탕 중심
preserve-font  → SVG 지정 글꼴 이름 최대한 유지
custom-map     → 프로젝트별 글꼴 매핑 파일
```

설정 기능을 추가할 때는 기본 동작을 바꾸지 않고, 기존 중국어·일본어·영문 경로와 충돌하지 않아야 합니다.

### P2 — 한국어 품질 검사

검토 가능한 항목:

- 한글 런인데 `lang="zh-CN"`으로 남은 경우 경고
- 한국어 글꼴이 없는 글꼴 스택 경고
- 지나치게 작은 한국어 본문
- 한 글자씩 부자연스럽게 분리된 `<tspan>`
- 한국어 문장 중간의 불필요한 공백
- 대체 글꼴로 예상되는 텍스트 오버플로

정적 검사만으로 디자인 품질을 단정하지 않고, 객관적으로 재현 가능한 오류만 자동화 대상으로 삼습니다.

## 유지보수 원칙

### 원본 동작 보존

한국어 지원은 기존 경로를 대체하지 않고 추가해야 합니다.

```text
영문 → en-US + Latin
중국어/CJK → zh-CN + EA
한국어 → ko-KR + Korean EA
```

### 작은 공통 헬퍼 변경

언어 감지와 글꼴 해석은 공유 헬퍼에 두고 호출부마다 복제하지 않습니다. 슬라이드 본문과 발표자 노트가 같은 판별 함수를 사용해야 합니다.

### 실제 PowerPoint 검수

XML이 유효한 것과 화면이 올바른 것은 다릅니다. 최소한 다음을 확인합니다.

- PowerPoint에서 파일 열림
- 한국어 검색·선택·수정 가능
- 글꼴 이름 확인
- 줄바꿈과 텍스트 상자 크기
- 애니메이션과 노트 보존
- 다시 저장한 뒤에도 문서 손상 없음

### 저장소 규칙 준수

이 저장소는 자동 테스트 파일을 두지 않는 규칙을 사용하므로, 변경 검증은 인라인 스모크 명령, 실제 프로젝트 변환, PR 설명의 재현 절차로 남깁니다.

### 포크 내부 유지

이 한국어 변경은 원본 저장소로 제출하지 않고 `Leafmire/ppt-master` 포크 내부 브랜치와 PR로 관리합니다. 원본 업데이트를 가져올 때 충돌 가능성이 높은 언어·폰트 헬퍼와 문서 링크를 우선 점검합니다.

## 범위 밖

다음 항목은 현재 한국어 포크의 핵심 목표가 아닙니다.

- PowerPoint를 완전히 대체하는 독립 편집기
- 모든 SVG/CSS 효과의 픽셀 단위 완전 재현
- 모든 상용 한국어 글꼴의 자동 임베딩
- 사용 권한이 없는 글꼴 또는 음성 배포
- 사람 검수 없는 사실 정확성 보장
- 원본 저장소와 독립적인 대규모 기능 포크

## 변경 제안 기준

새 기능은 다음 질문을 통과해야 합니다.

1. 한국어 사용자의 실제 실패를 해결하는가
2. 기존 언어 동작을 깨뜨리지 않는가
3. PowerPoint 편집성을 유지하는가
4. 코드와 문서가 함께 갱신되는가
5. 재현 가능한 스모크 검증 절차가 있는가
6. 포크에서 장기 유지 가능한 범위인가

## 상태 확인

가장 정확한 현재 상태는 다음 순서로 확인합니다.

1. `Leafmire/ppt-master`의 현재 브랜치와 커밋
2. 열려 있는 포크 내부 PR
3. `docs/ko/` 문서
4. 원본 저장소의 최신 로드맵과 릴리스

로드맵은 코드보다 앞서 기능을 보장하지 않습니다. 구현되지 않은 항목은 방향 또는 검토 대상입니다.
